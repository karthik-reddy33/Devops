# Linux Basics for DevOps Engineer

This file contains the Linux topics you should learn from the basics to become a DevOps engineer.

## 1. Linux Fundamentals
- What is Linux and why it is used in DevOps
- Difference between Linux distributions (Ubuntu, CentOS, RHEL)
- Basic Linux architecture: kernel, shell, filesystem
- Common directories: /etc, /home, /var, /tmp, /opt, /usr

## 2. Command Line Basics
Learn these basic commands:
- `pwd`, `ls`, `cd`, `mkdir`, `rm`, `cp`, `mv`
- `cat`, `less`, `head`, `tail`, `grep`, `find`
- `touch`, `chmod`, `chown`, `sudo`

## 3. File and Directory Management
- Create, edit, copy, move, and delete files
- Understand absolute and relative paths
- File permissions and ownership
- Symbolic links and hard links

## 4. Users, Groups, and Permissions
- Create users and groups
- Change passwords
- Understand read, write, execute permissions
- Use `chmod`, `chown`, `usermod`, `useradd`
- Basic sudo configuration

## 5. Package Management
- Install, update, and remove packages
- Learn package managers:
  - `apt` for Ubuntu/Debian
  - `yum` / `dnf` for RHEL/CentOS
- Understand repositories and dependency management

## 6. Process and Service Management
- View running processes with `ps`, `top`, `htop`
- Stop and start processes
- Manage services with `systemctl`
- Understand daemons and background jobs

## 7. Networking Basics
- IP addressing and subnetting basics
- `ping`, `curl`, `wget`, `ssh`, `scp`
- Check ports and connections
- Understand DNS, hosts, and routing basics

## 8. File Systems and Storage
- Disk usage with `df`, `du`
- List disks with `lsblk`
- Mount and unmount storage
- Understand partitions and filesystem types

## 9. Logs and Troubleshooting
- Read system logs
- Use `tail -f` for live logs
- Understand `/var/log`
- Use `journalctl` for systemd-based systems

## 10. Shell Scripting
- Variables, conditions, loops
- Functions and arguments
- Input/output redirection
- Pipes and filters
- Use `awk`, `sed`, and `grep` in scripts

## 11. Automation and Scheduling
- Cron jobs
- Task scheduling basics
- Writing small automation scripts

## 12. Security Basics
- SSH security basics
- Firewall basics
- Secure file permissions
- Understand sudo and root access
- Basic SELinux/AppArmor concepts

## 13. DevOps-Relevant Linux Topics
These topics are very important for DevOps engineers:
- SSH access and remote server management
- Shell scripting for automation
- Managing services and processes
- Linux networking and troubleshooting
- Log monitoring and debugging
- Container basics (Docker runs on Linux)
- Basic cloud server management

## 14. Recommended Learning Order
1. Linux commands and file handling
2. Permissions and users
3. Package management
4. Processes and services
5. Networking and SSH
6. Shell scripting
7. Logs and troubleshooting
8. Automation and cron
9. Security basics
10. Docker and cloud server basics

## 15. Minimum Skills You Should Know
Before moving to advanced DevOps tools, you should be comfortable with:
- Navigating the Linux filesystem
- Editing files from terminal
- Managing users and permissions
- Installing software
- Running and debugging services
- Writing simple shell scripts
- Connecting to remote machines via SSH

This is the foundation required for learning Git, Docker, Jenkins, Kubernetes, Terraform, and cloud platforms.
